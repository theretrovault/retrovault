# RetroVault Discord and Community Recommendation

This document defines the recommended Phase 1 community shape for RetroVault.

The goal is not to build a huge community overnight. The goal is to create a clean, credible support and feedback venue that matches a serious open-source self-hosted tool.

---

## Phase 1 community goal

Give RetroVault a public place where people can:
- ask setup/install questions
- report bugs and confusing behavior
- share screenshots/collections/use cases
- hear about releases and important changes

Recommended primary real-time surface:
- Discord

Recommended primary async/project surface:
- GitHub Issues + optionally GitHub Discussions

---

## Recommended Discord structure

### Core channels
- `#welcome`
  - what RetroVault is
  - where to start
  - links to docs/GitHub/install guide
- `#announcements`
  - releases, major fixes, important operator notes
- `#support`
  - install and troubleshooting help
- `#feature-ideas`
  - user suggestions and wishlist items
- `#bug-reports`
  - community-facing bug intake before or alongside GitHub Issues
- `#showcase`
  - screenshots, collections, setups, dashboards, field-mode stories
- `#general`
  - lighter conversation around collecting / the app / adjacent workflow talk

### Nice-to-have later
- `#release-testing`
- `#docker-and-selfhosting`
- `#conventions-and-hunts`
- `#api-and-integrations`

---

## Suggested roles

Keep it simple at first:
- Admin
- Maintainer
- Tester
- Community

Optional later:
- Early Access / Nightly Tester
- Contributor

---

## Recommended support model

### Discord is good for
- quick install help
- clarifying operator confusion
- lightweight feedback loops
- social proof and screenshots

### GitHub is better for
- actionable bugs
- tracked feature work
- release-linked regressions
- documentation changes

Recommended operating rule:
- Discord for conversation
- GitHub for durable tracked work

## Suggested welcome copy

Welcome to RetroVault.

RetroVault is a self-hosted retro game collection manager for collectors, flippers, and hunters. If you want help installing it, upgrading it, backing it up, or figuring out whether a bug is real, you’re in the right place.

Start here:
- GitHub: source, issues, releases
- Installation guide
- Developer/architecture docs
- Support and bug-reporting guidance

If you hit a problem, tell us:
- how you installed it
- what version/tag you’re running
- what you expected
- what happened instead
- screenshots or logs if you have them

---

## Suggested announcements copy pattern

Each announcement should ideally include:
- release/version
- why it matters in one sentence
- top 3-5 changes
- operator-impact notes (if any)
- upgrade/install/docs links

---

## Suggested support expectations copy

When asking for help, include:
- RetroVault version/tag
- install method (Docker/native/other)
- environment (Unraid, Debian, Pi, VPS, etc.)
- what broke
- exact error text if available
- screenshot/log snippet if useful

This dramatically reduces back-and-forth.

## Recommendation

For Phase 1, keep the community footprint intentionally small and high-signal:
- Discord for real-time support + lightweight community
- GitHub Issues for bugs
- optionally GitHub Discussions for async Q&A if Alex wants a searchable project-native forum

Do not overbuild moderation/process until real user traffic justifies it.
