## Recommended GitHub Discussions posture

### Confirmed Phase 1 direction
Enable GitHub Discussions publicly, and keep them secondary to:
- GitHub Issues for bugs
- docs for install/operator guidance

Discord is intentionally deferred for now because community remains GitHub-first in Phase 1.

### Recommended GitHub Discussions categories
- Q&A
- Ideas
- Show and Tell
- Releases / Announcements

This keeps GitHub searchable and project-native without pretending it needs to replace real-time support.
